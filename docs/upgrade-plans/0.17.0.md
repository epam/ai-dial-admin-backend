# Upgrade Plan — 0.17.0

> Auto-generated from `docs/INFRA-CHANGELOG.md` — review before applying.

## AI-DIAL Admin Backend (0.17.0)

This release adds new configuration-management environment variables, changes the default RBAC role mapping, and removes several settings — including the legacy `allowedRoles` variables (replaced by `roles-mapping`) and two metrics page-size variables. Review the **Removed environment variables** section carefully — removed settings are no longer honored.

---

## 1. New Environment Variables

| Setting | Default | Required (Yes/No) | Description |
| :--- | :--- | :--- | :--- |
| `CORE_CONVERSATIONS_METADATA_DEFAULT_LIMIT` | `256` | No | Default limit on the number of items in the conversations metadata response from DIAL Core. |
| `CONVERSATIONS_IMPORT_CONSECUTIVE_ERRORS_THRESHOLD` | `2` | No | Maximum number of consecutive errors allowed during conversations import. |

## 2. Changed Defaults

> [!IMPORTANT]
> The default for `config.rest.security.default.roles-mapping` changed from `{}` to a non-empty mapping. If you previously relied on the empty default together with the now-removed `allowedRoles` settings, you must verify the new mapping aligns with your identity provider before upgrading, or access may be denied.

| Setting | Default | Required (Yes/No) | Description |
| :--- | :--- | :--- | :--- |
| `config.rest.security.default.roles-mapping` | `{"ConfigAdmin":["FULL_ADMIN"],"admin":["FULL_ADMIN"]}` | No | Default changed from `{}`. Maps identity provider roles to application roles (`FULL_ADMIN`, `READ_ONLY_ADMIN`). |

**Helm `values.yaml`:**

```yaml
backend:
  env:
    # Override only if your IdP roles differ from the new default:
    config.rest.security.default.roles-mapping: '{"ConfigAdmin":["FULL_ADMIN"],"admin":["FULL_ADMIN"]}'
```

## 3. Removed Environment Variables

#### Removed environment variables

| Variable | Description |
| :--- | :--- |
| `config.rest.security.default.allowedRoles` | Removed. Was used to grant admin access by listing allowed identity provider roles. Use `config.rest.security.default.roles-mapping` instead, and configure that mapping before upgrading, otherwise affected users receive `403`. |
| `providers.*.allowed-roles` | Removed. Was used for per-provider allowed roles. Use `providers.*.roles-mapping` instead, which maps each provider's roles to application roles. |
| `METRICS_INFLUX2_DEFAULT_PAGE_SIZE` | Removed. Previously set the default page size for InfluxDB 2 metrics queries. The service now applies its built-in default page size. |
| `METRICS_INFLUX3_DEFAULT_PAGE_SIZE` | Removed. Previously set the default page size for InfluxDB 3 metrics queries. The service now applies its built-in default page size. |

**Helm `values.yaml` — remove and replace:**

```yaml
backend:
  env:
    # Remove these (no longer supported):
    # config.rest.security.default.allowedRoles: "ConfigAdmin,admin"
    # providers.azure.allowed-roles: "AzureRole1,AzureRole2"
    # METRICS_INFLUX2_DEFAULT_PAGE_SIZE: ...
    # METRICS_INFLUX3_DEFAULT_PAGE_SIZE: ...

    # Use instead:
    config.rest.security.default.roles-mapping: '{"ConfigAdmin":["FULL_ADMIN"],"admin":["FULL_ADMIN"]}'
    providers.azure.roles-mapping: '{"AzureRole1":["FULL_ADMIN"],"AzureRole2":["READ_ONLY_ADMIN"]}'
```
