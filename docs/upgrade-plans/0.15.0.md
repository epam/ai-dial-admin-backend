### 2.3 Observability ŌĆö Enabling InfluxDB 3 Mode

To switch the admin backend from InfluxDB 2 to InfluxDB 3, set `METRICS_DATASOURCE_TYPE` to `influx3` and configure the InfluxDB 3-specific env vars:

**Helm `values.yaml` ŌĆö InfluxDB 3 mode:**

```yaml
backend:
  env:
    METRICS_ENABLED: "true"
    METRICS_DATASOURCE_TYPE: "influx3"
    METRICS_STORAGE_HOST: "https://influxdb3.example.com:8086"
  secrets:
    METRICS_STORAGE_TOKEN: "<your-influxdb3-token>"
```

> **Note:** `METRICS_STORAGE_DATABASE` (default: `analytics-realtime`), `METRICS_STORAGE_ANALYTICS_TABLE` (default: `analytics`), `METRICS_STORAGE_MCP_ANALYTICS_TABLE` (default: `mcp_analytics`), and `METRICS_INFLUX3_DEFAULT_PAGE_SIZE` (default: `100`) can be omitted if defaults are acceptable. `METRICS_STORAGE_ORG`, `METRICS_STORAGE_ANALYTICS_MEASUREMENT`, and `METRICS_STORAGE_MCP_ANALYTICS_MEASUREMENT` are InfluxDB 2-only and have no effect in InfluxDB 3 mode.
